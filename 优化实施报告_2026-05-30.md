# SalesClaw 系统优化报告

> 日期：2026-05-30

---

## 概述

本次优化围绕三个核心方向展开：**功能引导优化**、**本体论前端对齐**、**导航拆分优化**，共涉及 7 个新建文件、15+ 个修改文件，TypeScript 编译零错误。

---

## 一、功能引导优化

### 1.1 问题诊断

| 编号 | 问题 | 影响 |
|------|------|------|
| G1 | 6个顶级导航过多，用户不知从何开始 | 认知负荷高 |
| G2 | 功能重叠：决策中枢/智能洞察/AI工具内容重复 | 用户困惑 |
| G3 | 无新手引导，首次登录直接面对复杂界面 | 新用户无所适从 |
| G4 | DecisionOverview 4个子Tab功能混杂 | 首页眼花缭乱 |
| G5 | 场景推演极其复杂，3层嵌套 | 技术门槛过高 |
| G6 | 无渐进式披露，所有功能一次性暴露 | 信息过载 |

### 1.2 实施方案

#### 导航重构：6 → 3+2

**优化前**：
```
决策中枢 | 知识图谱 | 场景推演 | 智能洞察 | AI 规则 | AI 对话 | 系统设置
```

**优化后**：
```
工作台 | 知识图谱 | AI 助手          ← 核心导航
───────────────────────────────────
▶ 高级功能（折叠）                   ← 按需展开
  ├ 场景推演
  └ 推理中心
```

- 系统设置移入用户头像下拉菜单
- 侧边栏底部添加"帮助"入口（可重新触发新手引导）

#### 新手引导组件

新建 `OnboardingGuide.tsx`，3步引导 Modal：

- **Step 1**：欢迎 + 平台简介
- **Step 2**：三大核心功能卡片（工作台/知识图谱/AI助手）
- **Step 3**：快捷操作引导（问AI/查看图谱/浏览待决策）

完成后记录 `localStorage`，不再显示。

#### 页面功能说明

新建 `PageHeader.tsx` 通用组件，每个页面顶部添加：
- 图标 + 标题 + 一行描述
- 可折叠的详细说明

#### 工作台重构

4个子Tab → 3个区域：

| 原Tab | 优化后 |
|-------|--------|
| 待决策 + 风险预警 | **今日概览**（合并+快捷入口） |
| 机会洞察 | **洞察流**（简化） |
| AI工具 | **AI 工具箱**（折叠展示） |

#### 场景推演简化

3个Tab → 2个Tab + 高级折叠：

- 默认只展示"场景模拟"
- "决策推演"和"反事实分析"折叠在"高级分析"按钮下

## 二、本体论前端对齐

### 2.1 问题诊断

后端本体论模型丰富（30+类型、6大域、完整生命周期、动作系统、事件流、时序数据），但前端仅展示了冰山一角。

| 编号 | 后端能力 | 前端现状 |
|------|---------|---------|
| O1 | 30+ 对象类型（6大域） | 图谱只展示6种图例 |
| O2 | 生命周期管理（5阶段） | 仅以标签展示，无状态流转 |
| O3 | 动作系统（参数+前置条件+副作用） | 按钮不可用 |
| O4 | 链接属性（strength/confidence/provenance/有效期） | 仅显示类型+目标名 |
| O5 | 时序数据（多组序列） | 数据映射错误，无法渲染 |
| O6 | 事件流 | 完全未展示 |
| O7 | 域模型属性（完整字段） | 仅4个硬编码字段 |
| O8 | 6大域分类 | 无域概念，所有对象平铺 |
| O9 | 合规风险等级 / 情感分析 | 不可见 |

### 2.2 实施方案

#### 域分类筛选 + 中文映射

- 6大域分组筛选器：核心域 / 收入目标管理 / 费用管理 / 客户管理 / 医学事务 / 合规管理
- 30+ 类型中文标签映射（Doctor→医生，Hospital→医院...）
- 图谱图例按域分组展示

#### 链接属性可视化

`RelatedLinksList.tsx` 增强：
- **强度(strength)**：蓝色进度条
- **置信度(confidence)**：颜色标签（高>0.8绿/中>0.5琥珀/低红）
- **来源(provenance)**：小标签（人工/自动/推理）
- **有效期(valid_from/valid_to)**：日期范围

#### 生命周期进度条

新建 `LifecycleProgress.tsx`：
```
潜在客户 → 开发中 → 成熟 → 风险 → 流失
  ●─────────●───────●──────○──────○
```
- 当前阶段高亮，已完成阶段绿色
- 风险/流失阶段红色背景 + "需要关注"提示

#### 动作执行面板

新建 `ActionExecuteModal.tsx`：
- 动作描述 + 审批提示
- 前置条件列表
- 动态参数输入表单（string/number/boolean）
- 副作用提示
- 异步执行 + 成功/失败反馈

#### 事件时间线

新建 `EventTimeline.tsx`：
- 垂直时间线 + 颜色编码事件类型
- 拜访(蓝) / 合规(红) / 告警(琥珀) / 动作(绿) / 备注(灰) / 状态变更(紫)
- 关联对象链接

#### 时序数据修复

`TimeSeriesDisplay.tsx` 重写：
- 修复数据映射（`TimeSeriesData` 格式）
- 按系列渲染 SVG sparkline（200×40px）
- 趋势箭头（上升绿/下降红/平稳灰）
- 超过5组系列可展开

#### GenericDetail 增强

自动展示所有属性 + 智能格式化：
- 数字：右对齐 + 本地化格式
- 数组：逗号分隔标签
- 布尔：绿色✓ / 红色✗
- 状态/生命周期/情感/合规标签

#### 专属详情组件

| 组件 | 新增展示属性 |
|------|-------------|
| DoctorDetail | 职称/科室/专科/处方权/影响力/处方量/拜访日期 |
| HospitalDetail | 准入状态/采购模式/年收入/床位数/位置 |
| ProductDetail | 品类/销售额/市场份额/价格 |

#### 合规/情感可视化

- 图谱节点：合规风险等级环（high红/medium橙）
- 图谱节点：情感指示点（positive绿/negative红/neutral灰）

#### 全局搜索增强

- 300ms 防抖后端 API 搜索
- 本地 + 后端结果合并去重
- 来源标签（本地/后端）

## 三、导航拆分优化

### 3.1 问题诊断

功能堆叠在页面内Tab中，不可见、不直接：
- 工作台3个内嵌Tab
- 推理中心2个Tab
- 场景推演3层嵌套

### 3.2 实施方案

将页面内嵌Tab拆分为独立左侧导航项：

```
优化前                         优化后
──────────                    ──────────
📊 工作台(3个内嵌Tab)          📊 工作台（纯概览）
🕸️ 知识图谱                   🕸️ 知识图谱
🤖 AI 助手                    ── 数据 ──
▶ 高级功能(折叠)               💡 智能洞察
  ├ 场景推演                   🧠 推理规则
  └ 推理中心(2个Tab)           ── 行动 ──
                               🎯 场景推演
                               🤖 AI 助手
                               🛠️ AI 工具箱
```

### 3.3 各页面内容

| 页面 | 内容 | 来源 |
|------|------|------|
| 工作台 | AI摘要 + 待决策 + 风险预警 + 快捷入口 | 原概览Tab |
| 智能洞察 | InsightEngine 完整功能 | 原推理中心-洞察Tab |
| 推理规则 | InferenceRulesPanel 完整功能 | 原推理中心-规则Tab |
| AI 工具箱 | 推理引擎状态 + 智能提醒 + Agent状态 | 原工作台-AI工具箱Tab |
